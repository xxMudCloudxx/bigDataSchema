## 3. ⚠️ Git 协作指南 (新手必看)

我们的项目依赖这份文档，**如果你修改了本地文件（例如 `API.md`），你必须立刻同步到 Git 仓库**，否则其他人无法工作。

请严格遵守以下 "五步工作流"：

### 五步工作流 (每次修改后都必须执行)

#### 第 1 步：(开始修改前) 同步最新版本
**永远**在修改前，先拉取服务器的最新版本，防止冲突。
```bash
git pull origin main
```



#### 第 2 步：(本地) 修改你的文件

例如，你（同学B）修改了 `API.md` 并添加了新的 API。

#### 第 3 步：(本地) 添加所有改动

```bash
git add .
```



#### 第 4 步：(本地) 提交改动 (写清楚你干了什么)



```Bash
git commit -m "B组：更新了 /api/query 接口的分页参数"
```

*(请**务必**写清楚 `-m` 消息，例如 "A组：更新了VM报告" 或 "C组：更新了算法API定义")*



#### 第 5 步：(本地) 推送到服务器



```Bash
git push origin main
```
